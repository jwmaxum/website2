권장 사항
현재 명세를 기반으로 개발을 진행하면 대부분의 기능을 구현할 수 있습니다. 다만 운영 효율성을 크게 높이는 핵심 요소로 다음 세 가지를 추가하는 것을 권장합니다.
1.	대시보드 위젯 사용자 설정: 관리자마다 자주 보는 카드(KPI, 주문, 문의 등)를 자유롭게 배치하거나 숨길 수 있도록 합니다. 
2.	공통 게시물 편집기: 공지, 보도자료, FAQ, 배너, 팝업 등은 하나의 에디터와 게시 워크플로우를 공유하도록 설계하여 유지보수성을 높입니다. 
3.	감사 로그 및 버전 복원: 모든 콘텐츠와 설정 변경은 이전 버전으로 복원할 수 있도록 저장하고, 변경 이력과 작업자를 추적할 수 있게 합니다. 
이 구조를 적용하면 기업 홈페이지와 쇼핑몰을 하나의 CMS에서 일관되게 운영하면서도 향후 다국어, 멀티 브랜드, 추가 서비스까지 확장하기 쉬운 기반을 갖출 수 있습니다



웹사이트 개발 계획서: Beauty of Joseon 참고 (Shop, Company 좌측 메뉴 구조)
1. 서론
본 문서는 사용자께서 제공해주신 '미인정선(Beauty of Joseon)' 웹사이트(https://beautyofjoseon.co.kr/ )를 참고하여, 'Shop' 및 'Company' 섹션을 좌측 메뉴 구조로 포함하는 웹사이트를 바이브코딩 방식으로 구축하기 위한 개발 계획을 제시합니다. 기존 프로젝트 문서와 참고 웹사이트 분석을 통해 구체적인 개발 로드맵을 수립하고, 각 단계별 목표와 기술 스택을 명시하여 효율적인 프로젝트 진행을 목표로 합니다.
2. 참고 사이트 분석: Beauty of Joseon (https://beautyofjoseon.co.kr/ )
미인정선 웹사이트는 깔끔하고 직관적인 사용자 인터페이스를 제공하며, 제품 중심의 쇼핑몰 기능과 브랜드 스토리를 효과적으로 전달하는 기업 정보 섹션이 조화롭게 구성되어 있습니다.
2.1. 주요 특징
•상단 메뉴: 'Shop', 'Brand', 'Collection', 'Event', 'Global' 등의 주요 카테고리로 구성되어 있으며, 검색, 계정, 장바구니 아이콘이 함께 배치되어 있습니다.
•메인 비주얼: 고품질의 이미지와 함께 주요 제품 또는 이벤트를 강조하는 슬라이드 배너가 특징입니다.
•제품 목록: 베스트셀러, 신제품 등 다양한 기준으로 제품을 분류하여 시각적으로 매력적인 방식으로 진열하고 있습니다.
•하단 푸터: 고객센터 정보, 소셜 미디어 링크, 이용약관, 개인정보처리방침 등 필수적인 정보가 체계적으로 정리되어 있습니다.
•반응형 디자인: 다양한 디바이스에서 최적화된 사용자 경험을 제공합니다.
2.2. 좌측 메뉴 구조 제안
사용자 요청에 따라, 미인정선 웹사이트의 'Shop' 및 'Company' 관련 콘텐츠를 좌측 메뉴 구조로 재구성하여 접근성을 높일 예정입니다.
3. 기존 프로젝트 문서 분석: default_site1 개발문서.pdf
제공된 default_site1 개발문서.pdf는 웹사이트의 백엔드 시스템, 특히 CMS(관리자 시스템)의 상세한 기능 명세와 데이터베이스 구조를 포함하고 있습니다. 이는 웹사이트의 견고한 백본을 구축하는 데 중요한 기반이 됩니다.
3.1. 데이터베이스 구조
문서에 명시된 주요 데이터베이스 테이블은 다음과 같습니다:
•Company: company_info, ceo_message, banners, footer
•Media: press, gallery, videos, downloads
•Products: categories, products, product_images
•Shop: members, carts, orders, order_items
•Customer: inquiries, inquiry_reply
•System: settings, menus, popup, admins
이 구조는 기업 정보, 미디어 자료, 제품 정보, 쇼핑몰 운영, 고객 관리 및 시스템 전반을 아우르는 포괄적인 데이터 관리를 지원합니다.
3.2. CMS(관리자 시스템) 기능
CMS는 기업 홈페이지와 쇼핑몰을 통합 운영하는 백오피스 역할을 수행하며, 다음과 같은 주요 관리 기능을 포함합니다:
•사이트 관리: 회사 정보, CEO 인사말, 회사 연혁, Footer, Contact, SNS, 이용약관, 개인정보처리방침 등 기업 홈페이지의 전반적인 내용을 관리합니다.

## 6.1. 쇼핑몰 & 자사몰 주요 구현 기능 (Front-end & Shopping Mall)

### 1) 토스페이먼츠(Toss Payments) 결제 연동 및 13대 DB 필드 수집
- **메인 결제 PG**: 토스페이먼츠(Toss Payments) 결제창 SDK 연동 완료.
- **결제수단 지원**: 신용/체크카드, 계좌이체, 가상계좌, 간편결제(토스페이, 카카오페이, 네이버페이 등).
- **Supabase DB (`public.payments` 테이블) 필수 13대 필드 자동 수집 및 저장**:
  1. `id` (UUID Primary Key)
  2. `order_id` (주문 식별자: e.g., `ORD-20260723-XXXX`)
  3. `user_id` (고객 ID/이메일)
  4. `pg` (`TOSSPAYMENTS`)
  5. `payment_key` (토스 고유 결제키)
  6. `transaction_id` (거래 식별자)
  7. `amount` (총 결제 금액)
  8. `vat` (부가세 10% 자동 계산)
  9. `status` (`DONE`, `CANCELLED`, `REFUNDED`)
  10. `method` (카드, 계좌이체 등)
  11. `approved_at` (승인 일시 TIMESTAMPTZ)
  12. `cancelled_at` (취소 일시 TIMESTAMPTZ)
  13. `refunded_amount` (환불 금액)

### 2) 고객 ID 배송지 수집 & 향후 재주문 자동 불러오기 (Address Mapping)
- **Supabase DB (`public.customer_addresses` 테이블) 및 로컬 DB 맵핑**:
  - 주문서 결제 시 입력받은 배송지 주소, 수령인 성명, 연락처를 고객 ID(이메일)와 자동 맵핑하여 DB에 저장합니다.
- **재주문 자동 채우기 (Auto-fill)**:
  - 재방문 고객이 로그인하거나 주문서에서 이메일을 입력 시 맵핑된 기본 배송지가 자동 불러와집니다.
- **마이페이지 배송지 관리**:
  - `/mypage` 회원 정보 관리 메뉴에서 고객 ID에 연동된 기본 배송지 주소를 확인하실 수 있습니다.

### 3) 국내 3대 택배사 (CJ대한통운, 로젠택배, 한진택배) API 실시간 배송 추적
- **실시간 API 연동 서비스 (`courierTrackingService.ts`)**:
  - `CJ대한통운` (코드 `04`), `로젠택배` (코드 `06`), `한진택배` (코드 `05`) 실시간 이동 경로 API 지원.
- **통합 배송 추적 모달 (`CourierTrackingModal.tsx`)**:
  - `집화완료` ➔ `Hub 이동중` ➔ `배송출발 (담당기사 연락처)` ➔ `배송완료` 4단계 실시간 타임라인 시각화.
- **관리자 & 회원/비회원 공용 조회**:
  - 관리자 주문 관리 대시보드(`/admin/orders`) 및 고객 마이페이지(`/mypage`) 모두에서 실시간 조회 가능.

### 4) 하이엔드 럭셔리 브랜드 스토리 페이지 (`/brand`) & 관리자 수정 기능
- **디자인 레이아웃**: Aesop, Sulwhasoo, Dior Beauty 스타일의 비디오 배경 풀스크린 럭셔리 디자인.

### 5) 대표이사 인사말 및 회사 정보 관리 (`/company?tab=ceo`)

- **구조화된 데이터**: Google 검색용 JSON-LD Schema.org 자동 생성 및 전역 헤더(`document.head`) 실시간 주입.

## 6.3. 데이터베이스 스키마 명세 (Supabase SQL Schema)

```sql
-- 1. Payments Table (토스페이먼츠 13대 필드)
CREATE TABLE IF NOT EXISTS public.payments (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_id TEXT NOT NULL,
    user_id TEXT NOT NULL,
    pg TEXT DEFAULT 'TOSSPAYMENTS',
    payment_key TEXT UNIQUE NOT NULL,
    transaction_id TEXT,
    amount NUMERIC NOT NULL,
    vat NUMERIC DEFAULT 0,
    status TEXT NOT NULL,
    method TEXT,
    approved_at TIMESTAMPTZ,
    cancelled_at TIMESTAMPTZ,
    refunded_amount NUMERIC DEFAULT 0,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 2. Customer Addresses Table (고객 ID 배송지 맵핑 DB)
CREATE TABLE IF NOT EXISTS public.customer_addresses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id TEXT NOT NULL,
    recipient_name TEXT NOT NULL,
    phone TEXT,
    address TEXT NOT NULL,
    is_default BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
```
